Cliente: Nome genérico das aplicações que integram as APIs dos parques. São clientes do parque.

Cliente não registado: É o programador da APISistemaCentral que pretende integrar as API dos parques no seu sistema e necessita de se registar na API para identificar as suas Reservas no parque.

Utilizador: É um utilizador humano registado na aplicação APISistemaCentral que pretende fazer reservas de Lugares de estacionamento.

Utilizador não registado: É um utilizador humano que pretende registar-se na APISistemaCentral para utilizar a aplicação.

API.Parque: Refere-se à API individual de cada parque público ou privado. Devolve uma lista de lugares de estacionamento disponíveis para aluguer, e permite efectuar reservas.

API.SistemaCentral: Refere-se à API que centraliza a gestão de Reservas em todas as APIParque. Devolve uma lista de Lugares de estacionamento disponíveis para aluguer, e permite efectuar Reservas e Sub-alugueres.

API.SubAluguer : Refere-se à API responsável por disponibilizar e permitir reservar os Lugares de estacionamento que os Utilizadores da APISistemaCentral pretendem sub-alugar.

Reserva: Identifica o aluguer de um Lugar de estacionamento num determinado Parque efectuado por um Cliente ou Utilizador.

Sub-aluguer: Um Utilizador efectua um aluguer de um Lugar disponibilizado e previamente reservado por outro Utilizador.

APP.Frontend: Aplicação (mobile e web) que funciona como frontend do Sistema Central e que permite aos utilizadores registados no sistema a utilização das funcionalidades disponibilizadas pelo mesmo
